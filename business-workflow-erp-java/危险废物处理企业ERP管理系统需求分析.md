危险废物处理企业ERP管理系统需求规格说明书
1 系统概述
1.1 系统定位
本系统是面向危险废物处理企业的一体化业务管理平台，以合同全流程管理为核心，融合客户关系管理、运输调度管理、仓储管理、财务管理、人员管理等功能模块，实现业务操作规范化、数据管理精细化、监管合规自动化、决策支持智能化，替代传统人工与Excel管理模式，降低运营风险，提升企业核心竞争力。
系统内部要做到数据留痕，防止篡改，合同及附件资料文件存档。系统内设计简单的审批流程，重要修改需要审批之后才能修改成功。
面向的系统使用者要简单、易用、高效、有特色、有差异性、有卖点。
要预留起码3个月的修改和磨合期。因为实际执行的过程细节需要打磨。这个用文字是表达不出来的，要实际经理过才知道怎样改。
1.2 核心目标
合规性目标：满足国家危险废物全过程信息化监管要求，实现危险废物从产生到处置信息可追溯。
效率提升目标：优化业务流程，实现部门间数据实时共享，将流程转为线上系统管理，提升合同执行与部门协同效率。
管理规范目标：建立标准化业务操作流程，实现数据修改留痕、权限精准控制，避免数据篡改风险，减少人工操作错误。
决策支持目标：通过可视化报表与数据分析功能，为企业经营决策提供数据支撑，实现风险提前预警与资源优化配置。

系统需基于各部门岗位职责，建立精准的权限分配与业务协同机制，确保数据在部门间高效流转，具体协同要求如下：
横向协同：实现市场部合同信息自动同步至生产部、财务部；生产部业务数据实时反馈至市场部与财务部；财务部结算信息同步至各相关部门。
纵向协同：支持管理层实时查看各部门业务进度与数据报表，实现指令快速下达与执行反馈。
定期推送：提示定期回访客户，续签合同，合同收运，合同收款，等等，到对应的职能人员，让他们跟进事项。

3 部门岗位职责规划（权限自定义系统）
3.1 市场部
3.1.1 市场商务岗
核心职能：市场开发、业务洽谈、业务投标
系统需求分析：
具备客户登记备案操作权限，支持手动录入或批量导入潜在客户信息，关联《国家危险废物名录》完成产废类型匹配。
使用客户报价管理功能，根据业务洽谈结果生成、调整报价单，跟踪报价单状态。
记录客户跟进日志，设置跟进提醒，同步业务投标相关资料至系统归档，支持投标进度跟踪。
查看所负责客户的合同执行进度、结算情况，接收相关业务预警通知。
3.1.2 市场总务岗
核心职能：配合商务岗出具 / 审核合同、协助经理发布市场公告、产废单位危废物平台操作、接单单位危险废物平台操作
系统需求分析：
具备合同创建、合规审核权限，基于商务岗提供的洽谈信息与报价单生成合同，执行合同合规性初审。
支持在系统内发布、管理市场公告，确保公告可追溯、可查询。
对接产废单位与接单单位的危险废物管理平台，实现数据同步录入、校验与上报，确保电子转移联单等信息合规。
协助整理合同归档资料，维护客户资质备案信息，接收资质到期预警并提醒相关人员。
3.2 生产部
3.2.1 调度
核心职能：接收订单→协调安排运输车辆
系统需求分析：
实时接收市场部同步的合同订单信息，自动生成调度任务单。
查看车辆实时状态（空闲、在途、维修），分配运输任务，手动调整调度计划并记录原因。
跟踪运输任务执行进度，接收异常情况反馈，协调处理调度过程中的问题。
关联仓管、司机、押运员信息，实现调度指令在线下发与确认。
3.2.4 仓管员
核心职能：进出货过磅+数量确认、控制仓库容量 + 协调出入库任务
系统需求分析：
对接运输模块数据，核对到库废物的类别、数量与运输单信息，记录结果。
实时更新仓库库存数据，查看库存预警信息，合理协调出入库任务，避免库存超期或容量不足。
生成入库单、出库单，关联处置订单信息，确保物料平衡可追溯。
3.3 财务部
3.3.1 核算岗
核心职能：出具结算单 / 对账单（与客户 / 供应商对账）、催收 / 支付款项、与业务员对账发提成
系统需求分析：
关联合同与生产业务数据，生成结算单、对账单，支持按合同类型（总价包干 / 按量结算）精准核算费用。
支持与客户 / 供应商在线对账，记录异议处理过程，审核结算单并推送至相关岗位。
接收应收应付款项到期提醒，记录催收 / 支付情况，跟踪款项到账状态。
按合同约定与业务员业绩数据，自动计算提成金额，生成提成核算表，与业务员对账确认。
3.3.3 总账岗
核心职能：核算
系统需求分析：
关联结算单、发票、付款凭证等数据，生成会计分录，支持手工调整与审核。

4.2 合同管理模块
4.2.1 合同创建与审核
支持创建总价包干、按量结算等多种合同类型。

4.2.2 合同执行跟踪
关联后续运输（可能多次运输）、仓储、财务等业务数据，更新合同执行进度，以百分比形式直观展示。
支持按合同编号查询全流程执行记录，包括订单详情、入库数据、结算情况等。
当合同执行出现延期、费用超支等异常情况时，需要相关人员上报至相关业务人员与管理层。
4.2.3 合同完结归档
当财务结算完成且所有业务环节结束后，人工确认。
合同归档需包含完整业务链条数据，包括合同原文、执行记录、结算凭证、客户反馈等，形成电子档案。
电子档案支持按合同类型、签订时间、客户名称等维度检索，归档后的数据不可修改，仅可查看。

4.4 财务管理模块
4.4.1 结算管理
基于合同约定与生产部业务数据，配置客户结算单，总价包干合同按合同金额结算，按量结算合同按实际处置量与单价计算。

### 收款回单管理（银行回单录入防错）
回单需录入字段（考虑ocr识别）：
回单编号（唯一标识）
客户全称
开户行
银行账号
收款金额
收款日期
上传银行电子回单扫描件
系统防错机制：回单编号唯一性校验（检测重复回单）
若回单编号重复，系统禁止提交，提醒“该回单已录入，可能重复入账”。
回单金额 ≠ 应收金额 的差异自动提示
多收 → 标记为“超收金额”，进入财务审核
少收 → 标记为“未收完”，自动形成剩余应收
结算单 ↔ 收款回单 自动匹配机制
系统需提供半自动匹配功能：
出纳选择结算单
录入/选择对应的收款回单
系统自动对比金额：
#### 匹配结果 1：金额相等 → 收款完成
自动记录收款完成时间
#### 匹配结果 2：回单金额 > 结算金额 → 超收
系统提示：“收款金额大于应收金额，是否将差额转为预收款？”并生成：超收记录
#### 匹配结果 3：回单金额 < 结算金额 → 少收
系统生成：未收完金额，该结算单保持“部分收款”状态，剩余应收金额继续保留
收款对账结果记录
所有收款匹配动作必须记录审计日志：
操作人
操作时间
匹配前金额
匹配后金额
回单编号
差额说明
保证资金流全链条可追溯
