### 2.2 DataSourceConfig

```java
/**
 * 数据源配置核心类
 * 
 * 功能：
 * 1. 初始化主库和从库 HikariCP 连接池
 * 2. 启动健康检查定时任务
 * 3. 处理故障转移逻辑
 * 4. 提供动态数据源路由
 */
@Configuration
@EnableConfigurationProperties(DataSourceProperties.class)
@ConditionalOnProperty(prefix = "spring.datasource.dbguardian", name = "enabled", havingValue = "true", matchIfMissing = true)
public class DataSourceConfig {

    @Autowired
    private DataSourceProperties properties;

    @Autowired
    private DatasourceCoordinationService coordinationService;

    private volatile DataSourceStatus currentStatus = DataSourceStatus.MASTER_ACTIVE;
    private volatile String currentMasterHost;
    private volatile int currentMasterPort;

    // 主库数据源
    @Bean(name = "masterDataSource")
    public DataSource masterDataSource() {
        HikariConfig config = new HikariConfig();
        config.setJdbcUrl(properties.getMaster().getUrl());
        config.setUsername(properties.getMaster().getUsername());
        config.setPassword(properties.getMaster().getPassword());
        config.setMaximumPoolSize(properties.getMaster().getHikari().getMaximumPoolSize());
        config.setMinimumIdle(properties.getMaster().getHikari().getMinimumIdle());
        config.setConnectionTimeout(properties.getMaster().getHikari().getConnectionTimeout());
        // ... 更多配置
        return new HikariDataSource(config);
    }

    // 从库数据源
    @Bean(name = "slaveDataSource")
    public DataSource slaveDataSource() {
        // 类似配置
    }

    // 动态路由数据源
    @Bean(name = "dataSource")
    @Primary
    public DataSource routingDataSource() {
        Map<Object, Object> targetDataSources = new HashMap<>();
        targetDataSources.put("master", masterDataSource());
        targetDataSources.put("slave", slaveDataSource());

        RoutingDataSource routingDataSource = new RoutingDataSource();
        routingDataSource.setTargetDataSources(targetDataSources);
        routingDataSource.setDefaultTargetDataSource(masterDataSource());
        return routingDataSource;
    }

    // 健康检查定时任务
    @PostConstruct
    public void initHealthCheck() {
        scheduledExecutorService.scheduleAtFixedRate(
            this::performHealthCheck,
            0,
            properties.getFailover().getCheckIntervalSeconds(),
            TimeUnit.SECONDS
        );
    }

    // 故障转移逻辑
    private synchronized void handleFailover() {
        if (!coordinationService.tryAcquireFailoverLock()) {
            return; // 已有其他实例在处理
        }
        try {
            // 1. 检查从库状态
            // 2. 提升从库为主库
            // 3. 更新路由
            // 4. 广播状态变更
            coordinationService.broadcastStatusChange(DataSourceStatus.SLAVE_PROMOTED);
        } finally {
            coordinationService.releaseFailoverLock();
        }
    }
}
```

### 2.3 DataSourceAspect

```java
/**
 * 读写分离 AOP 切面
 * 根据方法名自动决定使用主库还是从库
 */
@Aspect
@Component
public class DataSourceAspect {

    private static final Set<String> READ_PREFIXES = Arrays.asSet(
        "select", "get", "query", "find", "count", 
        "list", "page", "search", "load", "check"
    );

    @Autowired
    private RoutingDataSource routingDataSource;

    @Around("execution(* com..*Mapper.*(..))")
    public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
        String methodName = joinPoint.getSignature().getName().toLowerCase();
        
        if (isReadOnlyMethod(methodName)) {
            routingDataSource.setMethodType(MethodType.READ);
        } else {
            routingDataSource.setMethodType(MethodType.WRITE);
        }
        
        return joinPoint.proceed();
    }

    private boolean isReadOnlyMethod(String methodName) {
        return READ_PREFIXES.stream().anyMatch(methodName::startsWith);
    }
}
```

### 2.4 DatasourceCoordinationService

```java
/**
 * Redis 分布式协调服务
 * 
 * 功能：
 * 1. 管理分布式锁，防止多实例同时故障转移
 * 2. 广播状态变更
 * 3. 订阅状态变更通知
 */
@Service
public class DatasourceCoordinationService {

    private static final String STATUS_KEY = "dbguardian:datasource:status";
    private static final String MASTER_INSTANCE_KEY = "dbguardian:datasource:master:instance";
    private static final String FAILOVER_LOCK_KEY = "dbguardian:failover:lock";
    private static final String STATUS_CHANNEL = "dbguardian:status:channel";

    @Autowired
    private StringRedisTemplate redisTemplate;

    /**
     * 尝试获取故障转移分布式锁
     */
    public boolean tryAcquireFailoverLock() {
        return Boolean.TRUE.equals(
            redisTemplate.opsForValue().setIfAbsent(
                FAILOVER_LOCK_KEY, 
                getInstanceId(), 
                Duration.ofSeconds(30)
            )
        );
    }

    /**
     * 释放故障转移锁
     */
    public void releaseFailoverLock() {
        String currentHolder = redisTemplate.opsForValue().get(FAILOVER_LOCK_KEY);
        if (getInstanceId().equals(currentHolder)) {
            redisTemplate.delete(FAILOVER_LOCK_KEY);
        }
    }

    /**
     * 广播状态变更
     */
    public void broadcastStatusChange(DataSourceStatus status) {
        Map<String, Object> message = new HashMap<>();
        message.put("instanceId", getInstanceId());
        message.put("status", status.name());
        message.put("timestamp", System.currentTimeMillis());
        
        redisTemplate.convertAndSend(STATUS_CHANNEL, message);
    }

    /**
     * 检查 Redis 连接是否正常
     */
    public boolean isHealthy() {
        try {
            return "PONG".equals(redisTemplate.getConnectionFactory()
                .getConnection().ping());
        } catch (Exception e) {
            return false;
        }
    }
}
```

## 3. 数据流

### 3.1 正常读操作流程

```
Service.selectXxx()
       ↓
MyBatis Mapper (DataSourceAspect 拦截)
       ↓
RoutingDataSource.setMethodType(READ)
       ↓
HikariPool-Slave.getConnection()
       ↓
Slave DB 执行查询
```

### 3.2 正常写操作流程

```
Service.insertXxx()
       ↓
MyBatis Mapper (DataSourceAspect 拦截)
       ↓
RoutingDataSource.setMethodType(WRITE)
       ↓
HikariPool-Master.getConnection()
       ↓
Master DB 执行写入 + 复制到 Slave
```

### 3.3 故障转移流程

```
1. 健康检查发现 Master 不可用
          ↓
2. DatasourceCoordinationService.tryAcquireFailoverLock()
          ↓ (成功获取锁)
3. DataSourceConfig.handleFailover()
          ↓
4. 检查 Slave 数据一致性
          ↓
5. 更新 Replication 配置 (change master to)
          ↓
6. SlaveDataSource → 新主库
          ↓
7. coordinationService.broadcastStatusChange(SLAVE_PROMOTED)
          ↓
8. 其他实例收到通知，同步状态
          ↓
9. DatasourceStatusListener 更新本地状态
```
