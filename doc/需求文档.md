# DBGuardian - Spring Boot 数据库高可用 Starter

## 1. 项目概述

**项目名称**: DBGuardian
**项目类型**: Spring Boot 数据库高可用自动配置 Starter
**核心价值**: 为 Spring Boot 应用提供**读写分离**、**自动故障转移**、**分布式协调**能力
**目标用户**: 企业级 Spring Boot 应用开发者

**当前状态**: 已完成从原项目的抽离，成为独立的 DBGuardian Spring Boot Starter

## 2. 核心功能

### 2.1 已实现功能

| 功能 | 描述 | 源文件 |
|-----|------|-------|
| **读写分离** | 自动路由读/写操作到不同数据源 | `DataSourceAspect.java` |
| **连接池** | HikariCP 高性能连接池 | `DataSourceConfig.java` |
| **故障转移** | 主库故障时自动将从库升为主库 | `DataSourceConfig.java` |
| **健康检查** | 每30秒定时检测主从库健康状态 | `DataSourceConfig.java` |
| **分布式协调** | Redis 实现多实例状态同步 | `DatasourceCoordinationService.java` |
| **状态广播** | 故障转移后通知其他实例 | `DatasourceStatusListener.java` |
| **降级启动** | 允许从库故障时降级运行 | `DataSourceConfig.java` |
| **原主库恢复** | 原主库恢复后作为从库追赶数据 | `DataSourceConfig.java` |

### 2.2 数据源状态

```java
public enum DataSourceStatus {
    MASTER_ACTIVE,   // 主库活跃
    SLAVE_PROMOTED,  // 从库已升主
    DEGRADED         // 降级模式
}
```

### 4.3 配置属性设计

```yaml
spring:
  datasource:
    dbguardian:
      # 主库配置
      master:
        url: jdbc:mysql://localhost:3306/test
        username: root
        password: password
        hikari:
          maximum-pool-size: 20
          minimum-idle: 5
          connection-timeout: 30000
          idle-timeout: 600000
          max-lifetime: 1800000
      # 从库配置
      slave:
        url: jdbc:mysql://localhost:3307/test
        username: root
        password: password
        hikari:
          maximum-pool-size: 20
          minimum-idle: 5
      # 故障转移配置
      failover:
        enabled: true
        check-interval: 30s        # 健康检查间隔
        auto-promote-slave: true    # 自动升主
      # 分布式协调配置
      coordination:
        redis:
          enabled: true
          channel: dbguardian:status:channel
          lock-key: dbguardian:failover:lock
      # 高级配置
      advanced:
        allow-degraded-startup: true  # 允许降级启动
```

## 5. 读写分离策略

### 5.1 自动路由规则

| 方法前缀 | 目标数据源 | 说明 |
|---------|-----------|------|
| `select*` | 从库 | 查询操作 |
| `get*` | 从库 | 获取操作 |
| `query*` | 从库 | 查询操作 |
| `find*` | 从库 | 查找操作 |
| `count*` | 从库 | 计数操作 |
| `list*` | 从库 | 列表操作 |
| `page*` | 从库 | 分页操作 |
| `search*` | 从库 | 搜索操作 |
| 其他 | 主库 | 写操作 |

### 5.2 故障转移时的行为

当检测到主库故障：
1. 获取分布式锁（防止多实例同时操作）
2. 将从库提升为主库
3. 广播状态变更
4. 所有操作切换到新主库
5. 释放分布式锁

## 6. 使用示例

### 6.1 添加依赖

```xml
<dependency>
    <groupId>io.dbguardian</groupId>
    <artifactId>dbguardian-spring-boot-starter</artifactId>
    <version>1.0.0</version>
</dependency>
```

### 6.2 配置文件

```yaml
spring:
  datasource:
    dbguardian:
      master:
        url: jdbc:mysql://master-host:3306/mydb
        username: root
        password: secret
      slave:
        url: jdbc:mysql://slave-host:3306/mydb
        username: root
        password: secret
      failover:
        enabled: true
```

### 6.3 自动使用

```java
@Service
public class UserService {

    // 读操作 - 自动路由到从库
    public User getUser(Long id) {
        return userMapper.selectById(id);  // select → 从库
    }

    // 写操作 - 自动路由到主库
    public void createUser(User user) {
        userMapper.insert(user);  // insert → 主库
    }

    // 故障转移时自动切换
    public void processOrder(Order order) {
        // 事务内自动使用主库
        orderMapper.insert(order);
    }
}
```
